# Deployment Guide

## Quick Start (VPS)

### 1. Prerequisites

- Docker Engine ≥ 24
- Docker Compose V2 (`docker compose`)
- A reverse proxy (Nginx Proxy Manager recommended) handling TLS termination
- Port 6969 open in your firewall (BitTorrent tracker)

### 2. Clone and configure

```bash
git clone https://github.com/yourname/hugging-bay .
cp .env.example .env
```

Edit `.env` with strong secrets:

```bash
# Generate JWT secret:
openssl rand -hex 32

# Generate Meilisearch key:
openssl rand -hex 16

# Set MINIO credentials to whatever you like
```

### 3. Build and launch

```bash
make build
make up
```

Check logs:

```bash
docker compose logs -f backend
docker compose logs -f frontend
```

### 4. Reverse proxy setup (Nginx Proxy Manager)

Add two proxy hosts in NPM:

| Domain | Forward Hostname | Forward Port | Notes |
|--------|-----------------|--------------|-------|
| `your-domain.com` | `localhost` | `3000` | Frontend — enable WebSocket support |
| `your-domain.com/api` | `localhost` | `8080` | Backend API |

Or configure a single proxy host that routes `/api/*` to port 8080 and everything else to port 3000 using a custom Nginx config snippet:

```nginx
location /api {
    proxy_pass http://localhost:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    client_max_body_size 512M;
}
location / {
    proxy_pass http://localhost:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

> **Important**: set `client_max_body_size 512M` (or higher) in your proxy to allow `.torrent` file uploads.

### 5. First admin account

Register normally via the UI, then promote yourself in the database:

```bash
docker compose exec postgres psql -U hb -d huggingbay -c \
  "UPDATE users SET is_admin=TRUE WHERE username='yourusername';"
```

Or use the Makefile shortcut:

```bash
make admin USERNAME=yourusername
```

### 6. Tracker URL

Set `TRACKER_ANNOUNCE_URL=udp://your-domain.com:6969/announce` in `.env`
so that magnet links generated by the backend include your tracker.

### Upload release (multipart/form-data)

```bash
curl -X POST /api/models/llama-3-70b/releases \
  -H "Authorization: Bearer $TOKEN" \
  -F 'metadata={"version":"1.0","sha256":"abc...","quantization":"Q4_K_M","parameter_count":70000000000,"file_manifest":[{"path":"model.gguf","size":40000000000}]}' \
  -F 'torrent=@llama-3-70b.torrent'
```

## Scaling

For production traffic:
- Run `backend` with multiple replicas behind your reverse proxy
- Use a managed PostgreSQL (e.g. RDS, Supabase)
- Use a managed Meilisearch (Meilisearch Cloud)
- Consider CloudFront or another CDN in front of the frontend
- MinIO can be replaced with S3 by changing `MINIO_ENDPOINT` and enabling SSL
