# Deployment Guide

## Quick Start (VPS)

### 1. Prerequisites

- Docker Engine ≥ 24
- Docker Compose V2 (`docker compose`)
- Port 80, 443, and 6969 open in your firewall

### 2. Clone and configure

```bash
git clone https://github.com/yourname/hugging-bay .
cp .env.example .env
```

Edit `.env` with strong secrets:

```bash
# Generate JWT secret:
openssl rand -hex 32

# Generate Meilisearch key:
openssl rand -hex 16

# Set MINIO credentials to whatever you like
```

### 3. Go dependency resolution (first time only)

The Go backend needs its `go.sum` generated before the first Docker build:

```bash
cd backend
go mod tidy   # requires Go 1.22+ installed locally
cd ..
```

> **Alternative (no local Go needed):** The Dockerfile runs `go mod tidy` automatically,
> so you can skip this step and let the first build handle it.

### 4. Build and launch

```bash
docker compose build
docker compose up -d
```

Check logs:

```bash
docker compose logs -f backend
docker compose logs -f frontend
```

### 5. HTTPS with Let's Encrypt (recommended)

Replace the nginx config with Certbot or use Caddy as a reverse proxy:

```bash
# Install Certbot
apt install certbot python3-certbot-nginx -y

# Get cert (stop nginx in Docker first or use standalone mode)
certbot certonly --standalone -d your-domain.com

# Copy certs into the nginx_certs Docker volume
cp /etc/letsencrypt/live/your-domain.com/fullchain.pem /var/lib/docker/volumes/hugging-bay_nginx_certs/_data/
cp /etc/letsencrypt/live/your-domain.com/privkey.pem   /var/lib/docker/volumes/hugging-bay_nginx_certs/_data/
```

Then update `nginx/nginx.conf` to add an HTTPS server block with:
```nginx
listen 443 ssl;
ssl_certificate     /etc/nginx/certs/fullchain.pem;
ssl_certificate_key /etc/nginx/certs/privkey.pem;
```

### 6. First admin account

Register normally via the UI, then promote yourself in the database:

```bash
docker compose exec postgres psql -U hb -d huggingbay -c \
  "UPDATE users SET is_admin=TRUE WHERE username='yourusername';"
```

### 7. Tracker URL

Set `TRACKER_ANNOUNCE_URL=udp://your-domain.com:6969/announce` in `.env`
so that magnet links generated by the backend include your tracker.

## API Reference

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/models` | — | List models (paginated) |
| GET | `/api/models/:slug` | — | Get model detail |
| POST | `/api/models` | ✓ | Create model |
| PUT | `/api/models/:slug` | ✓ | Update model |
| GET | `/api/models/:slug/releases` | — | List releases |
| POST | `/api/models/:slug/releases` | ✓ | Create release (multipart) |
| GET | `/api/models/:slug/releases/:version/magnet` | — | Get magnet URI |
| GET | `/api/models/:slug/releases/:version/torrent` | — | Download .torrent |
| GET | `/api/models/:slug/releases/:version/manifest` | — | Signed manifest JSON |

| GET | `/api/latest` | — | Recently added |
| GET | `/api/popular` | — | Most downloaded |
| POST | `/api/auth/register` | — | Create account |

| GET | `/api/me/api-keys` | ✓ | List API keys |

### Upload release (multipart/form-data)

```bash
curl -X POST /api/models/llama-3-70b/releases \
  -H "Authorization: Bearer $TOKEN" \
  -F 'metadata={"version":"1.0","sha256":"abc...","quantization":"Q4_K_M","parameter_count":70000000000,"file_manifest":[{"path":"model.gguf","size":40000000000}]}' \
  -F 'torrent=@llama-3-70b.torrent'
```

## Scaling

For production traffic:
- Run `backend` with multiple replicas behind nginx upstream
- Use a managed PostgreSQL (e.g. RDS, Supabase)
- Use a managed Meilisearch (Meilisearch Cloud)
- Consider CloudFront or another CDN in front of nginx
- MinIO can be replaced with S3 by changing `MINIO_ENDPOINT` and enabling SSL
